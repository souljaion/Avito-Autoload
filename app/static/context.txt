## 1. ЦЕЛЬ ПРОЕКТА И БИЗНЕС-КОНТЕКСТ

Avito Autoload — веб-приложение для управления товарными объявлениями на Avito
(крупнейший российский маркетплейс). Система позволяет:

- Импортировать товары с Avito или создавать вручную
- Управлять каталогом: категории, фото, описания, цены
- Генерировать XML-фиды в формате Avito Autoload v3
- Загружать фиды через Avito API (OAuth2)
- Мониторить отчёты Avito (applied/declined)
- Собирать аналитику (просмотры, контакты, избранное) с маркерами эффективности
- Планировать публикации по расписанию (inline и массовое)
- Работать с моделями и фотопаками для мультиаккаунтных схем
- Автоимпорт новых объявлений каждые 3 часа
- Автодетекция проданных товаров
- Синхронизировать фото из CRM (отдельная БД)

Бизнес-модель: ресейл одежды/обуви/аксессуаров через несколько аккаунтов Avito.
Один товар (модель) может публиковаться на разных аккаунтах с уникализацией фото.

## 3. БАЗА ДАННЫХ — ВСЕ ТАБЛИЦЫ

16 таблиц. Подключение: postgresql+asyncpg://avito_user:avito_pass@localhost:5433/avito_autoload

### accounts
  Аккаунты Avito с OAuth2 credentials.
  Поля: id, avito_user_id (BigInt), name, client_id, client_secret,
        access_token, token_expires_at, phone, address, autoload_enabled,
        report_email, schedule, avito_sync_minute (Integer, nullable),
        extra (JSONB), created_at, updated_at
  avito_sync_minute — минута каждого часа когда Авито забирает фид (например 17 для Zulla, 48 для Parker)

### products
  Товары — центральная сущность.
  Поля: id, avito_id (BigInt, unique), account_id (FK→accounts),
        model_id (FK→models, SET NULL), sku,
        brand, model, category, subcategory, goods_type, goods_subtype,
        size, color, material, condition, price (Integer, рубли),
        title, description, use_custom_description (Boolean, default False),
        status (String 20),
        scheduled_at, published_at, scheduled_account_id (FK→accounts),
        image_url, extra (JSONB), created_at, updated_at
  use_custom_description: False → XML использует шаблон аккаунта, True → product.description

### product_images
  Поля: id, product_id (FK CASCADE), url, filename, sort_order, is_main, created_at

### models
  Модели — шаблоны для группировки вариантов товара.
  Поля: id, name, brand (String 100), description,
        category (String 100), subcategory (String 100),
        goods_type (String 100), goods_subtype (String 100), created_at
  Категория модели копируется в товар при создании через create-one/create-all.

### photo_packs
  Поля: id, model_id (FK CASCADE), name, created_at

### photo_pack_images
  Поля: id, pack_id (FK CASCADE), file_path, url, sort_order, created_at

### account_description_templates
  Шаблоны описаний для аккаунтов.
  Поля: id, account_id (FK, unique), description_template (Text)

### pack_usage_history
  История использования фотопаков на аккаунтах.
  Поля: id, pack_id (FK), account_id (FK), uniquified (Boolean), created_at
  Первое использование — без уникализации, повторное — с уникализацией.

### listings
  Поля: id, product_id (FK CASCADE), account_id (FK CASCADE),
        status (draft/scheduled/published), scheduled_at, published_at,
        avito_id (BigInt), created_at, updated_at

### listing_images
  Поля: id, listing_id (FK CASCADE), file_path, order, created_at

### feed_exports
  Поля: id, account_id (FK), file_path, products_count, status, uploaded_at, upload_response (JSONB), created_at

### autoload_reports
  Поля: id, account_id (FK), avito_report_id, status, total_ads, applied_ads, declined_ads, extra (JSONB), created_at

### autoload_report_items
  Поля: id, report_id (FK CASCADE), ad_id, avito_id, url, status, messages (JSONB), error_text, created_at

### avito_categories
  Поля: id, avito_id, slug, name, parent_id (FK CASCADE, self-ref), show_fields, fields_data (JSONB), synced_at

### item_stats
  Статистика (КУМУЛЯТИВНЫЕ значения Авито). Дельта за период = MAX - baseline.
  Поля: id, product_id (FK CASCADE), listing_id (FK SET NULL), avito_id (BigInt),
        views, contacts, favorites, price, captured_at
  Уникальный индекс: (product_id, DATE(captured_at))

## 5. СТАТУСЫ ТОВАРОВ

  imported  — импортирован с Авито (не показывается в /products)
  draft     — создан через систему, черновик
  scheduled — в очереди на публикацию (время задаётся на /schedule)
  active    — активен на Авито
  published — опубликован через автозагрузку
  paused    — приостановлен
  sold      — продан (автодетекция)

XML-фид включает товары со статусом active И scheduled.
is_ready_for_feed() НЕ проверяет статус — только обязательные поля.
Если use_custom_description=False — description берётся из шаблона аккаунта.
Condition дефолт: "Новое с биркой". Максимум 10 фото в фиде.


## 6. СИСТЕМА МОДЕЛЕЙ И ФОТОПАКОВ

Модель хранит: name, brand, category, goods_type, subcategory, goods_subtype.
При создании товара (create-one/create-all) копируются все поля + condition + brand.

Фотопаки: первое использование на аккаунте — без уникализации,
повторное — с уникализацией (crop/brightness/noise). Отслеживается через pack_usage_history.


## 7. XML-ФИДЫ

Формат: Avito Autoload XML v3. Генерация через lxml.
Включает active + scheduled товары. Description может браться из шаблона аккаунта.
Загрузка: OAuth2 Client Credentials → upload_feed (multipart).
Публикация запланированных: каждые 5 минут, обновляет product.status=active.


## 8. БЕЗОПАСНОСТЬ

HTTP Basic Auth. Публичные пути: /health, /feeds/*.xml, /media/*, /static/*.
Fetch-патч в base.html: meta[name=auth] с base64 credentials → Authorization header.


## 9. APScheduler — 6 ФОНОВЫХ ЗАДАЧ

  stats_sync        — каждые 3 часа
  publish_scheduled — каждые 5 минут
  image_sync        — каждые 30 минут
  sold_detection    — каждые 6 часов
  token_refresh     — каждые 50 минут
  avito_import      — каждые 3 часа


## 10. ОСОБЕННОСТИ

- Время в БД: naive UTC. Отображение: Europe/Moscow.
- avito_sync_minute: минута когда Авито забирает фид → показ ~HH:MM в UI.
- item_stats views — кумулятивные. Дельта = MAX - baseline. 1 snapshot = unknown.
- Описания: use_custom_description=False → шаблон аккаунта, True → product.description.
